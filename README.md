# 沪深300 PE+PB估值动态投资优化系统

## 目录结构

```

├── fetch_data.py     # 数据抓取（akshare）— 含 PE、PB
```

## 日常使用（每月一次）

在 VS Code 中打开 `analysis.ipynb`，从头到尾运行所有格子即可。

输出：
- 当前 PE/PB 百分位、合成估值分位、趋势状态和建议仓位
- 策略 vs 基准的绩效对比
- 净值曲线 / 仓位 / 估值分位图表
- reports/ 目录下的 CSV 和摘要文本

## 策略逻辑（v2）

### 1. 双因子合成估值分位

分别计算 PE 和 PB 在过去 1200 个交易日内的历史百分位，取均值作为**合成估值分位**。

> 相比单用 PE，加入 PB 可以避免"周期股景气顶峰时 PE 虚低"的陷阱。

### 2. 线性仓位映射

| 合成估值分位 | 建议仓位 |
|------------|--------|
| ≤ 20%      | 100%   |
| 20% ~ 80%  | 线性插值（从100%降至0%）|
| ≥ 80%      | 0%     |

相比硬分档，仓位变化更平滑，减少边界附近频繁调整。

### 3. 趋势过滤

收盘价低于 250 日均线时，无论估值多低，仓位强制归零。

> 防止在下跌趋势中抄底，等待趋势回暖再介入。

## 参数优化（Walk-forward）

```bash
python optimize.py
```

- **全样本网格搜索**：找出历史夏普最优参数组合（Top 10）
- **Walk-forward 验证**：7年训练 / 3年验证，检验参数样本外有效性，并输出训练→验证夏普衰减幅度

衰减过大（> 0.5）说明参数可能过拟合，谨慎使用。
